화면이 의도대로 나오지 않을 때, AI 에게 **추측이 아니라 브라우저 화면을 직접 보고** 진단·수정하도록 지시할 수 있습니다. 프로젝트에 설치된 `debug-ui` 스킬과 `chrome-devtools-mcp` 플러그인이 이 흐름을 자동으로 처리합니다.

## debug-ui 스킬은 무엇을 해주나

스킬이 트리거되면 AI 가 다음을 자동으로 수행합니다:

1. **브라우저 페이지 확보** — `localhost:3000` 에 접속, 화면이 렌더될 때까지 대기
2. **스크린샷 + 접근성 스냅샷** — 사람이 보는 모습과 화면 구조를 함께 캡처
3. **콘솔 / 네트워크 점검** — 에러 메시지, 실패한 API 요청을 정독
4. **원인 한 줄 보고 → 수정** — 추측 패치 금지. 실제 신호를 본 뒤에만 코드 수정
5. **수정 후 재캡처** — 변경이 반영됐는지 다시 확인하고 결과 보고

## 화면 확인 요청

💬 Claude Code 대화창에 자연어로 요청하면 스킬이 알아서 트리거됩니다.

| 상황                | 프롬프트 예시                                      |
| ------------------- | -------------------------------------------------- |
| 화면 확인           | "지금 화면 어떻게 보여?"                           |
| 화면이 깨져 보일 때 | "현재 페이지를 확인하고 깨진 부분을 수정해줘"      |
| 콘솔/네트워크 에러  | "브라우저에 에러가 있는지 확인하고 수정해줘"       |
| PRD/ADR 일치 검증   | "PRD F1 대로 됐는지 시각적으로 비교해줘"           |
| 모바일 반응형 점검  | "모바일 사이즈에서는 어떻게 보여?"                 |
| 클릭/입력 검증      | "결제 버튼을 누르면 어떻게 되는지 실제로 눌러봐줘" |

::alert["화면 봐줘", "스크린샷 찍어줘", "지금 이상하게 보이는데 뭐가 문제야?" 같은 짧은 요청만으로도 충분합니다. 스킬이 알아서 화면을 확보하고 진단합니다.]{type="info"}

## 사전 조건

이 스킬이 동작하려면 두 가지가 필요합니다 (둘 다 워크숍에서 이미 준비되어 있습니다):

1. **개발 서버가 실행 중** — 별도 터미널 탭에서 `pnpm dev:web` 으로 띄워둔 상태. AI 는 dev 서버를 직접 띄우지 않습니다.
2. **chrome-devtools 플러그인 connected** — `/plugin` Installed 탭에서 `chrome-devtools-mcp` 가 `✔ enabled` 인지 확인. 안 되어 있으면 [MCP 서버 확인](/getting-started/verify-mcp) 페이지를 참고하세요.

## PRD/ADR 일치 검증의 가치

스킬에 _"PRD F1 대로 됐는지 시각적으로 비교해줘"_ 라고 요청하면 AI 가 다음과 같이 표로 정리해 보여줍니다:

```
| 항목            | 기대 (PRD/ADR)    | 현재 화면  | 결과 |
| --------------- | ----------------- | ---------- | ---- |
| 결제 버튼 라벨  | "주문하기"        | "주문하기" | ✅   |
| 옵션 선택 UI    | 드롭다운          | 카드형     | ❌   |
| 메인 색상       | #2563eb (primary) | #1d4ed8    | ⚠️   |
```

데모 직전에 한 번 돌리면 **고객 앞에서 어긋난 항목을 모르고 시연하는 사고** 를 방지할 수 있습니다.

## 일반적인 문제 해결

### 브라우저에 변경사항이 반영되지 않을 때

브라우저 캐시를 강제로 새로고침합니다:

- **Mac**: `Cmd + Shift + R`
- **Windows**: `Ctrl + Shift + R`

그래도 반영되지 않으면 💻 개발 서버 탭(탭 1)에서 `Ctrl+C` 로 서버를 종료한 뒤 💻 터미널에서 재시작합니다:

:::code{showCopyAction=true showLineNumbers=false language=bash}
pnpm dev:web
:::

### 포트 충돌 오류

포트 3000이 이미 사용 중인 경우, 💻 터미널에서 `PORT` 환경변수를 지정해서 다른 포트로 실행할 수 있습니다.

::::tabs
:::tab{label="Mac"}

:::code{showCopyAction=true showLineNumbers=false language=bash}
PORT=3001 pnpm dev:web
:::

:::
:::tab{label="Windows (PowerShell)"}

:::code{showCopyAction=true showLineNumbers=false language=powershell}
$env:PORT=3001; pnpm dev:web
:::

:::
::::

그 후 브라우저에서 `http://localhost:3001` 로 접속하고, AI 에게 _"디버깅은 3001 포트로 해줘"_ 라고 한 번 알려주면 됩니다.

### 스킬이 트리거되지 않을 때

`/plugin` Installed 탭에서 `chrome-devtools-mcp` 가 `✔ enabled` 인지 확인하고, 그래도 안 되면 `/reload-plugins` 를 한 번 실행합니다.
