## 사전 준비: 터미널 여는 법

이 워크숍에서 말하는 "터미널" 은 아래와 같습니다.

### Windows — PowerShell 여는 법

1. 작업 표시줄 **시작(⊞)** 버튼 클릭, 또는 키보드의 **Windows 키** 입력.
2. `PowerShell` 이라고 입력.
3. 검색 결과에서 **"Windows PowerShell"** 또는 **"PowerShell"** 을 **우클릭 → "관리자 권한으로 실행"** 선택.
4. 파란색(또는 검은색) 창이 열립니다. 이 창이 💻 **터미널** 입니다.

### Mac — 터미널 여는 법

1. 키보드에서 **⌘(Command) + Space** 를 눌러 Spotlight 를 엽니다.
2. `터미널` 이라고 입력하고 Enter.
3. 검은색 창이 열립니다. 이 창이 💻 **터미널** 입니다.

> **Tip**: 이 창을 **절대 닫지 마세요**. 오늘 내내 이 창에서 명령을 입력하게 됩니다. 창을 닫으면 다시 열어도 되지만, Claude Code 대화 내역이 사라집니다.

## 2. Claude Code 설치하기 (13:20 ~ 14:30)

1일차에는 **Claude Code 하나만** 설치하고, Bedrock 자격증명을 Claude Code 에 등록하는 것까지 진행합니다. 프로젝트 clone 이나 복잡한 환경 설정은 다음 워크숍에서 진행합니다.

### 2-1. 원라이너 설치 (Mac / Windows 공통)

필요한 도구(Node.js, Claude Code) 를 한 번에 설치합니다. 💻 터미널 또는 💻 PowerShell 에 **아래 한 줄을 그대로 복사해서 붙여넣고 Enter** 를 누르세요.

**Mac**:

```bash
# 💻 터미널 (Mac)
curl -fsSL https://raw.githubusercontent.com/haandol/ui-poc-workspace/main/scripts/install-claude-code.sh | bash
```

**Windows** — 💻 PowerShell 을 **관리자 권한** 으로 열고:

```powershell
# 💻 PowerShell (Windows, 관리자 권한)
iwr -useb https://raw.githubusercontent.com/haandol/ui-poc-workspace/main/scripts/install-claude-code.ps1 | iex
```

설치는 약 3~5분 소요됩니다. 진행 로그가 빠르게 올라가는 것이 정상입니다. **"Claude Code is ready"** 와 유사한 메시지가 나오면 완료.

> **설치가 도중에 멈추거나 오류 메시지가 나타나면**: 도우미에게 편하게 요청해주세요. 원라이너가 내부적으로 실행하는 것은 (1) Homebrew/winget 확인, (2) Node.js LTS 설치, (3) `npm install -g @anthropic-ai/claude-code` 세 단계입니다.

설치 확인 — 💻 터미널 또는 💻 PowerShell 에서:

```bash
# 💻 터미널 / PowerShell 공통
claude --version
```

버전 번호가 출력되면 설치 성공입니다.

### 2-2. Claude Code 실행 및 자격증명 입력

다음 섹션 §4 에서 본격적으로 대화를 시작합니다. 여기서는 Claude Code 를 한 번 실행해 **진행자가 제공한 Bedrock 자격증명(API Key 등)** 을 등록해둡니다.

**Mac**:

```bash
# 💻 터미널 (Mac)
mkdir -p ~/Desktop/claude-play && cd ~/Desktop/claude-play && claude
```

**Windows**:

```powershell
# 💻 PowerShell (Windows)
$desktop = [Environment]::GetFolderPath('Desktop')
New-Item -ItemType Directory -Path "$desktop\claude-play" -Force | Out-Null
cd "$desktop\claude-play"
claude
```

Claude Code 대화창이 열리면(💬 프롬프트 커서가 깜박임), 아래 슬래시 명령을 입력합니다.

```
💬 Claude Code 대화창
/setup-bedrock
```

그러면 Claude 가 자격증명 입력을 안내합니다. **진행자가 공유한 링크에서 API Key 를 복사** 해 요청하는 값을 차례대로 붙여넣으세요.

일반적으로 아래 값들을 묻습니다 (진행자 안내에 따라 실제 항목은 조금 다를 수 있습니다):

1. **AWS_ACCESS_KEY_ID** — 진행자 링크에서 복사
2. **AWS_SECRET_ACCESS_KEY** — 진행자 링크에서 복사
3. **AWS_SESSION_TOKEN** (Workshop Studio 를 사용하는 경우) — 진행자 링크에서 복사
4. **AWS_REGION** — 보통 `us-east-1`

입력이 끝나면 Claude 가 확인 메시지를 보여줍니다. 이후 간단한 테스트 질문을 해서 응답이 오는지 확인해보세요:

```
💬 Claude Code
안녕, 잘 들리면 한 줄로 답해줘.
```

응답이 오면 자격증명 등록 완료입니다. Claude Code 창은 **`/exit` 로 닫지 말고 그대로 열어두세요** — §4 에서 계속 사용합니다.

## 4. Claude Code 첫 실행 (14:50 ~ 15:10)

§2-2 에서 열어둔 Claude Code 대화창에서 이어서 진행합니다. 이 섹션을 마치면 Claude Code 에 기본 질문과 파일 생성 요청을 할 수 있게 됩니다.

> **중요**: 이 섹션 동안 Claude Code 창을 닫지 말고, **같은 창에서 계속 대화를 이어갑니다**. 여러 번 `claude` 를 새로 실행하면 이전 대화 맥락이 사라집니다.
>
> Claude Code 창을 닫으셨다면 §2-2 의 명령으로 다시 실행해주세요.

### 4-1. 기본 대화 해보기

아래 문장을 **하나씩** 💬 Claude Code 대화창에 입력해봅니다.

```
💬 Claude Code — 질문 1
지금 내가 있는 폴더에 뭐가 있어?
```

```
💬 Claude Code — 질문 2
Three.js 가 뭔지 한 단락으로 설명해줘.
```

```
💬 Claude Code — 질문 3
hello.txt 라는 파일을 만들고 "안녕" 이라고 한 줄 적어줘.
```

질문 3을 입력하면 **파일 생성 승인 요청** 이 뜹니다. 다음 섹션에서 조작 방법을 자세히 살펴봅니다.

### 4-2. 승인 요청과 조작 방법

Claude 가 **파일을 수정하거나 명령을 실행하려 할 때** 승인 요청이 뜹니다. 아래 형태와 비슷한 화면이 나타납니다:

```
Do you want to proceed?
❯ 1. Yes
  2. Yes, and don't ask again
  3. No, tell Claude what to do differently
```

- **1번 Yes** 를 고르면 해당 작업을 실행합니다. (Enter 또는 `1`)
- **3번 No** 를 고르면 실행을 막고 추가 지시를 입력할 수 있습니다. (`3` 또는 Esc)
- **2번** 은 "앞으로 이런 종류는 물어보지 말라" 는 뜻. 처음에는 누르지 않기를 권장합니다.

앞선 질문 3의 승인 요청에서 **1번 Yes** 를 선택해보세요. 작업 폴더에 `hello.txt` 가 생긴 것을 파일 탐색기 / Finder 에서 확인할 수 있습니다.

### 4-3. 자주 쓰는 단축키

- `/help` — 💬 대화창 안에서 입력. 사용 가능한 명령과 단축키를 보여줍니다.
- `Esc` — 💬 대화창 안에서 누름. AI 가 답변 중일 때 **중단** 하고 새로운 지시를 입력할 수 있습니다.
- `/exit` 또는 Ctrl+C 두 번 — 💬 대화창을 종료하고 터미널로 돌아갑니다.

## 5. 3D 도시 만들어보기 (15:10 ~ 16:10)

Three.js 로 돌아가는 3D 도시를 **단일 `index.html` 파일** 로 만듭니다. 빌드 도구도 서버도 필요 없고, 완성된 파일은 **더블클릭으로 브라우저에서 바로 열립니다**.

완성되면 화면에 아래 같은 모습이 나타납니다.

- 격자 도로와 수십~수백 개의 건물이 있는 도시
- 마우스 드래그로 시점 회전, 스크롤로 줌
- 좌측 상단 패널에서 **건물 수, 건물 높이, 밀도, 밤/낮, 창문 색상** 을 실시간 변경

### 5-1. 첫 요청 — 원샷으로 3D 도시 만들기

§4 에서 열어둔 💬 Claude Code 대화창에서 이어서 진행합니다. 아래 프롬프트를 **전체 그대로** 붙여넣으세요. 길지만 한 번에 붙여넣어도 됩니다.

```
💬 Claude Code — 원샷 프롬프트

바탕화면에 city-lab 이라는 새 폴더를 만들고, 그 안에 단일 index.html 파일 하나로 동작하는 Three.js 3D 도시 데모를 만들어줘.
- Mac 이면 ~/Desktop/city-lab, Windows 면 %USERPROFILE%\Desktop\city-lab 에 만들어야 해.
- 빌드 도구, 서버, 외부 에셋 없이 index.html 을 더블클릭하면 브라우저에서 바로 돌아가야 해.

=== 로딩 방식 ===
- Three.js 는 CDN + importmap 으로 불러올 것.
  three@0.164.x 와 three/addons/ (OrbitControls 포함) 사용.

=== 장면 구조 ===
- 맵 크기: 대략 mapSize = 180, halfMap = 90.
- 격자 도로: roadIndices = [-3, -2, -1, 0, 1, 2, 3], 각 도로 좌표 = roadIdx × 24 (roadStep = 24).
  - 가로 도로(x축 방향)와 세로 도로(z축 방향)가 동일하게 배치되어 격자를 이룬다.
  - 도로 폭(roadW) = 6.
- 건물:
  - Three.js 기본 BoxGeometry 만 사용해 폭/깊이/높이를 랜덤.
  - 위치는 격자 교점이 아닌 블록 안쪽 랜덤 좌표, 단 도로(roadW/2 안쪽) 위에는 올라가지 않게 체크.
  - seed 기반 난수(선형 합동 또는 mulberry32) 로 생성해서 슬라이더를 움직여도 기존 건물 위치가 흔들리지 않게.
- 도로 중앙에 짧은 노란색 차선 마커(길이 ~1.5, 간격 ~3) 를 주기적으로 배치.
- 도로 양쪽 가장자리에 얇은 연회색 보도블록 라인(curb) 표시.

=== 밤/낮 두 가지 모드 ===
- 각 모드는 배경, 안개색, 주변광, 헤미스피어 라이트, 주광(sun), 건물 색, 이미시브 등 전체 라이팅 프리셋을 한 번에 바꾼다.
- 밤 모드 기준 톤:
  - 배경/안개: #121829 (짙은 네이비, 너무 까맣지 않게)
  - AmbientLight 0.75, HemisphereLight (#3f4f78 / #141826) 0.95, DirectionalLight sun 0.7 (#9dc3ff 푸른 톤)
  - 건물: 기본색 #545d75 / #6c7690 두 가지를 랜덤으로, 이미시브 #222a42 × 0.4
  - 도로 바탕: #333a4d, 차선: #f6d27a, 보도블록: #5a6578
  - 수직 그라디언트 스카이돔: 위쪽 #070a16 → 중간 #131a30 → 지평선 #2a2040
  - 달 (반지름 10) 우측 상단 (120, 120, 60) 에 크림색 디스크 + 푸른 글로우 2겹(반지름 24, 42, AdditiveBlending)
  - 달 방향의 DirectionalLight 1.2 추가해 건물에 방향성 하이라이트 부여
  - 별 600개 BufferGeometry + PointsMaterial 로 먼 구면에 뿌리기
  - 각 건물 창문: 작은 Plane (0.45×0.45) 을 건물 4면에 격자로 올리고 랜덤으로 약 45% 만 켜기. 창문 색은 Accent 색 따라감.
- 낮 모드 기준 톤:
  - 배경: #cfdce8, 안개 #d7e1ea
  - AmbientLight 0.7, HemisphereLight (#dfe8f2 / #b5beca) 0.6, sun 1.3 (#fff4e0)
  - 건물: #e7ebf0 / #c0c7d0, 이미시브 0
  - 창문은 opacity 0 (꺼짐), 달/별/스카이돔 숨김.

=== 컨트롤 패널 (좌상단 고정) ===
반투명 어두운 카드(블러 백드롭) 에 다음 UI 를 배치.
1. Buildings 슬라이더: 20~240, 기본 120. 값 바뀌면 도시 재생성.
2. Max Height 슬라이더: 4~30, 기본 12.
3. Density 슬라이더: 0.3~1.0, 기본 0.85 (낮은 건물 비율).
4. Mood 토글 버튼: "Night" / "Day" (기본 Night).
5. Accent 스와치 4개: 파랑 #6ea8ff, 주황 #ffb26a, 보라 #b893ff, 민트 #7dffd4.
   스와치 변경 시 건물 창문 색이 즉시 바뀐다.
- 폰트는 Inter/Pretendard/system-ui 계열, 글자 간격은 넓게 (letter-spacing 0.18em 정도) 대문자.
- 하단 중앙에는 "Drag to orbit · Scroll to zoom · Right-drag to pan" 힌트 텍스트.

=== 카메라 & 조작 ===
- PerspectiveCamera fov 42, 초기 위치 (70, 60, 90), target (0, 2, 0)
- OrbitControls, damping 활성, maxPolarAngle 은 지면 아래를 보지 못하게 제한.
- 줌은 허용하되 너무 가깝거나 멀지 않게 (min 30, max 260).
- 그림자(shadowMap PCFSoft) 활성.

=== 기타 ===

#### 실험 2. 횡단보도와 사람들 걸어다니게 하기

```
💬 Claude Code — 실험 2
도시에 횡단보도와 걸어다니는 사람 40명을 추가해줘.

[횡단보도]

[사람 모양]
- 사람은 Three.js Group 하나로, 아래 두 부품 조합.
  - 몸통: BoxGeometry 0.3 × 1.2 × 0.3, y=0.6 위치. 색은 아래 팔레트에서 랜덤.
  - 머리: SphereGeometry 반지름 0.18, y=1.35 위치. 색은 피부톤 #f0d6be 고정.
- 색상 팔레트 6종: peach #e7c3b4, sage #bfd4b2, lilac #c5c5e3, wheat #e9d39d, dusty blue #b0c8d4, rosy #d9b6c4.
- 밤 모드에서 어둡게 묻히지 않도록 몸통/머리 모두 각자 색 × 0.18 정도를 emissive 로 주고 emissiveIntensity 0.75.
  낮 모드에서는 emissiveIntensity 0 으로 초기화.

[배치와 이동]
- 40명 각각 초기 상태:
  - 축(axis): 가로 도로(x축) / 세로 도로(z축) 중 랜덤.
  - 어느 도로인지(roadIdx): 격자 좌표 중 랜덤.
  - side: 해당 도로 기준 좌/우 인도(+1 / -1) 중 랜덤.
  - dir: 걷는 방향 +1 / -1 랜덤.
  - baseSpeed: 1.1 ~ 1.9 중 랜덤.
  - 시작 위치는 해당 도로 길이(mapSize) 범위 안에서 랜덤.
- 인도(sidewalk) 모드:
  - 도로 폭 / 2 + 0.6 만큼 도로 중심에서 바깥쪽으로 떨어진 선을 따라 걷는다.
  - 맵 끝을 넘으면 반대쪽 끝에서 다시 등장(wrap-around). world 바깥으로는 절대 나가지 않게 해.
- 횡단(crossing) 모드:
  - 인도를 걷다가 가장 가까운 교차로 좌표와 1.2 이내로 가까워졌고 쿨다운이 지났을 때만 건너기 시작.
  - 건너기 시작할 때 위치를 해당 교차로 좌표로 스냅하고, 속도를 baseSpeed × 1.6 으로 올린 뒤 반대쪽 인도(-side) 를 향해 일직선으로 이동.
  - 목표 인도 좌표에 도달하면 side 를 반대로 갱신하고 다시 인도 모드로 복귀. 복귀 후 3~8초 쿨다운.
- 횡단 중 사람의 방향(rotation.y) 은 실제 이동 방향을 향하도록 갱신.

[안정성]
- 순간이동/사라짐 방지가 중요해. 사람을 먼 교차로로 순간이동시키지 말고, 오직 자기가 걷고 있는 도로의 "가장 가까운 교차로" 에서만 건너기 시작하도록.
- 모든 좌표는 roadIdx × roadStep 기준으로만 계산하고, 매 프레임마다 world 경계를 체크해서 벗어나지 않게.
```

#### 실험 3. 고질라가 나타나 건물을 부수기

```

[고질라 모양]
- Three.js 기본 지오메트리(BoxGeometry, CylinderGeometry, SphereGeometry) 조합으로 하나의 Group 에 묶어.
- 기본 크기(scale 적용 전):
  - 몸통: BoxGeometry 3 × 4 × 2 (폭 × 높이 × 깊이), y=4

- 파괴 애니메이션(약 1.2초):
  - 건물의 scale.y 를 1 → 0 으로 선형 감소.
  - 동시에 x/z 방향으로 ±0.15 정도 좌우 흔들림(sin 파).
  - 완료되면 해당 메쉬를 city 그룹에서 제거하고 geometry·material 모두 dispose.
- 건물이 완전히 쓰러지는 순간 해당 위치에 **먼지 입자**(Three.Points, 반지름 ~0.05, 회색 #a0a0a0, 30~50개) 를 추가.
  먼지는 위로 퍼지면서 1초 안에 opacity 가 0 으로 페이드 아웃된 뒤 제거.

[하늘 변화]
- 고질라가 활성화된 동안 하늘 색을 붉은 톤으로 부드럽게 전환:

  - 배경/안개 색도 #2a0e12 로 lerp.
  - 고질라가 사라지면 원래 밤/낮 톤으로 복귀.
- 전환은 0.8초 정도에 걸쳐 부드럽게.

[컨트롤 & 상태 관리]

  - ON → OFF: 고질라를 씬에서 제거하고 dispose. 진행 중이던 건물 파괴 애니메이션은 그대로 완료.
- 슬라이더로 도시를 재생성할 때 고질라 상태도 함께 리셋(Kaiju 버튼도 기본 OFF 로).

[안정성]
- 부서진 건물 배열을 매 프레임 깨끗이 관리해서 null/undefined 참조가 생기지 않게.
- 고질라와 거리 계산 시 y 좌표는 무시하고 XZ 만 사용.
- dispose 누락이나 메모리 누수가 없도록 파괴 완료 시 geometry·material 확실히 정리.
```

각 단계에서 AI 가 **내 요청을 어떻게 해석했는지** 를 관찰하세요. 같은 프롬프트라도 사람마다 결과물이 다르게 나옵니다.

백엔드 없이 모든 데이터와 API 를 모킹해서 웹 UI 만 프로토타입할 거야.

내가 만들고 싶은 서비스는 중소기업 대표를 위한 주간 회의 준비 도우미 야.

아래 내용을 포함해서 리서치해줘:
1. 해당 서비스 도메인의 주요 경쟁사 분석 (핵심 기능, UX 특징, 강점/약점)

리포트는 한글로 작성해줘.
```

> **대괄호 `[ ]` 는 반드시 지우고** 본인 아이디어로 교체하세요. 대괄호째 입력하면 결과가 부정확해집니다.
