# Non-Tech UI PoC Workshop Runbook — Day 1

비개발자(PM)를 위한 AI 기반 UI PoC 워크숍 **1일차** 런북입니다.
1일차의 목표는 **(1) Claude Code 설치 완료, (2) 3D 헤드셋 카탈로그 데모로 AI 와 대화하는 감각 익히기, (3) Amazon Quick Suite 의 Quick Research 로 리서치 PDF 확보** 입니다.
다음 워크숍(PRD 작성 → UI PoC 개발) 은 약 1주일 뒤에 진행되며, 오늘 받아둔 리서치 PDF 를 재료로 시작합니다.

> **핵심 안내**: 이 워크숍에서는 AI 도구와 **대화하듯이** 작업합니다.
> 진행 중 궁금한 점이 있으면 언제든 AI 에게 질문하세요.
> 예: "이 폴더가 뭐 하는 거야?", "지금 뭐가 설치된 거야?", "이 에러 메시지 풀어서 설명해줘"

## 사전 준비: 터미널 여는 법

이 워크숍에서 말하는 "터미널" 은 아래와 같습니다. **처음 사용하는 분도 바로 따라 할 수 있습니다.**

### Windows — PowerShell 여는 법

1. 작업 표시줄 **시작(⊞)** 버튼 클릭, 또는 키보드의 **Windows 키** 입력.
2. `PowerShell` 이라고 입력.
3. **"Windows PowerShell"** 또는 **"PowerShell"** 이 검색 결과에 나타나면 **우클릭 → "관리자 권한으로 실행"** 을 선택.
4. 파란색(또는 검은색) 창이 열립니다. 이 창이 💻 **터미널** 입니다.

### Mac — 터미널 여는 법

1. 키보드에서 **⌘(Command) + Space** 를 눌러 Spotlight 를 엽니다.
2. `터미널` 이라고 입력하고 Enter.
3. 검은색 창이 열립니다. 이 창이 💻 **터미널** 입니다.

> **Tip**: 이 창을 **절대 닫지 마세요**. 오늘 내내 이 창에서 명령을 입력하게 됩니다. 창을 닫으면 다시 열어도 되지만, Claude Code 대화 내역이 사라집니다.

## 2. Claude Code 설치하기 (13:20 ~ 14:30)

설치 스크립트를 실행하면 모든 환경 설정이 자동으로 완료됩니다. 완료 후 작업 폴더는 `바탕화면/ui-poc-workspace/` 에 생성됩니다.

### 2-1. 설치 스크립트 실행

**Mac** — 💻 터미널 에 아래 명령을 **한 줄로** 붙여넣고 Enter:

```bash
# 💻 터미널 (Mac)
curl -fsSL https://raw.githubusercontent.com/haandol/ui-poc-workspace/main/scripts/bootstrap.sh | bash
```

**Windows** — 💻 PowerShell 에 아래 명령을 **순서대로** 붙여넣고 각각 Enter:

```powershell
# 💻 PowerShell (Windows) — 1단계: 프로젝트 내려받기
$desktop = [Environment]::GetFolderPath('Desktop')
git clone https://github.com/haandol/ui-poc-workspace "$desktop\ui-poc-workspace"
cd "$desktop\ui-poc-workspace"
```

```powershell
# 💻 PowerShell (Windows) — 2단계: 설치 스크립트 실행
powershell -ExecutionPolicy Bypass -File scripts\setup.ps1
```

설치는 약 5~10 분 소요됩니다. 진행 로그가 빠르게 올라가는 것이 정상입니다.

> **스크립트가 실패했다면**: 도우미에게 바로 문의하거나, [수동 설치 가이드](./INSTALLATION.md) 를 참고하세요. 20분 이상 혼자 끌지 마세요.

### 2-2. MCP 연결 상태 확인

설치가 완료되면 **같은 터미널** 에서 작업 폴더로 이동한 뒤 Claude Code 를 실행합니다.

```bash
# 💻 터미널 (Mac)
cd ~/Desktop/ui-poc-workspace && claude
```

```powershell
# 💻 PowerShell (Windows)
cd "$([Environment]::GetFolderPath('Desktop'))\ui-poc-workspace"; claude
```

> **처음 `claude` 를 실행하면** 로그인 안내가 나타날 수 있습니다. 진행자 안내에 따라 로그인/자격증명을 입력하세요.

Claude Code 대화창이 열리면(💬 프롬프트 커서가 깜박임), 아래 명령을 입력합니다:

```
💬 Claude Code 대화창
/mcp
```

`pdf-reader`, `alps-writer`, `asset-generator` 등 MCP 서버 목록이 표시되면 정상입니다. `✔ connected` 가 여러 개 보이면 됩니다.

### 2-3. 이미지 에셋 생성 설정하기 (선택)

AI 이미지 생성 기능을 사용하려면 FAL API Key 를 설정합니다. 진행자에게 전달받은 키를 환경변수로 등록합니다.

**먼저 Claude Code 대화창을 종료해야 합니다.** 💬 대화창에서 `/exit` 를 입력하거나 Ctrl+C 를 두 번 누르면 터미널로 돌아옵니다.

```bash
# 💻 터미널 (Mac)
echo 'export FAL_KEY="진행자에게_전달받은_키"' >> ~/.zshrc
source ~/.zshrc
```

```powershell
# 💻 PowerShell (Windows) — 설정 후 PowerShell 창을 닫고 다시 엽니다
[Environment]::SetEnvironmentVariable('FAL_KEY', '진행자에게_전달받은_키', 'User')
```

설정 후 Claude Code 를 다시 실행하면 `/mcp` 에서 `asset-generator` 가 `✔ connected` 상태로 변경됩니다.

> **참고**: FAL API Key 가 없어도 워크숍 진행에는 문제가 없습니다. 이미지 에셋 생성 기능만 비활성화됩니다.

### 2-4. 진행 상태 추적 설정하기

워크숍 진행 상황을 진행자와 공유하기 위해 Airtable 연동을 설정합니다. 진행자가 전달한 **Airtable API Key** 를 준비한 뒤, Claude Code 대화창에 아래 명령을 입력합니다.

```
💬 Claude Code 대화창
/workshop-status setup
```

Claude 가 두 가지를 물어봅니다:

1. **Airtable API Key** — 진행자에게 전달받은 값 (`pat_...` 로 시작)
2. **닉네임** — 진행 상태에 표시될 이름 (예: "PM 김민수", "기획팀 이영희")

설정이 완료되면 이후 단계(3D 헤드셋 카탈로그, Quick Research 등)에서 **진행 상태가 자동으로 공유** 됩니다.

> **참고**: Airtable 키가 없어도 워크숍 진행에는 문제가 없습니다. 진행 상태 추적 기능만 비활성화됩니다.

## 4. Claude Code 첫 실행 (14:50 ~ 15:10)

본격 실습 전에 Claude Code 의 기본 조작을 익힙니다. 이 섹션은 **모두 💬 Claude Code 대화창에서 입력** 합니다.

> **중요**: 이 섹션 동안 Claude Code 창을 닫지 말고, **같은 창에서 계속 대화를 이어갑니다**. 여러 번 `claude` 를 새로 실행하면 이전 대화 맥락이 사라집니다.

### 4-1. 기본 대화 해보기

작업 폴더에서 💻 터미널에 `claude` 를 입력해 대화창을 연 상태로, 아래 문장을 **하나씩** 입력해봅니다.

```
💬 Claude Code — 질문 1
지금 이 폴더가 어떤 프로젝트인지 한 줄로 요약해줘.
```

```
💬 Claude Code — 질문 2
docs 폴더에 어떤 파일들이 있는지 알려줘.
```

```
💬 Claude Code — 질문 3
package.json 을 읽고 어떤 프로젝트인지 설명해줘.
```

> **관찰 포인트**: 질문 1과 질문 3의 답변 상세도가 다릅니다. **AI 에게 "어디를 보라고" 알려주면 답이 구체적** 이라는 감각이 1일차의 핵심입니다.

### 4-2. 승인 요청과 조작 방법 익히기

Claude 가 **파일을 수정하거나 명령을 실행하려 할 때** 승인 요청이 뜹니다. 아래 형태와 비슷한 화면이 나타납니다:

```
Do you want to proceed?
❯ 1. Yes
  2. Yes, and don't ask again
  3. No, tell Claude what to do differently
```

- **1번 Yes** 를 고르면 해당 작업을 실행합니다. (Enter 또는 `1`)
- **3번 No** 를 고르면 실행을 막고 추가 지시를 입력할 수 있습니다. (`3` 또는 Esc)
- **2번** 은 "앞으로 이런 종류는 물어보지 말라" 는 뜻. 처음에는 누르지 않기를 권장합니다.

**실험**: 아래를 입력해서 승인 요청을 한 번 경험해봅니다.

```
💬 Claude Code
hello.txt 라는 파일을 만들고 "안녕" 이라고 한 줄 적어줘.
```

승인 요청이 뜨면 **1번 Yes** 로 허용. 작업 폴더에 `hello.txt` 가 생긴 것을 파일 탐색기에서 확인해보세요.

### 4-3. 자주 쓰는 단축키

- `/help` — 💬 대화창 안에서 입력. 사용 가능한 명령과 단축키를 보여줍니다.
- `Esc` — 💬 대화창 안에서 누름. AI 가 답변 중일 때 **중단** 하고 새로운 지시를 입력할 수 있습니다.
- `/exit` 또는 Ctrl+C 두 번 — 💬 대화창을 종료하고 터미널로 돌아갑니다.

> **Tip**: 이 조작을 지금 몸에 익혀두면 다음 워크숍에서 **"그만해", "되돌려줘"** 같은 반응을 빠르게 할 수 있습니다.

## 5. 3D 헤드셋 카탈로그 만들기 데모 (15:10 ~ 16:10)

"AI 에게 어디까지 맡길 수 있는가" 를 직접 실험하는 시간입니다. 제공된 헤드셋 제품 이미지를 사용해, **단일 `index.html` 파일** 로 가상 오디오 브랜드의 3D 헤드셋 카탈로그 페이지를 만듭니다. 빌드 도구도 서버도 필요 없고, 완성된 파일은 **더블클릭으로 브라우저에서 바로 열립니다**.

**오늘 만들 결과물 이미지**:

- 중앙에 헤드셋 제품 이미지가 3D 공간에서 천천히 회전
- 마우스 드래그로 각도 조정 가능
- 하단에 브랜드명 · 제품명 · 가격 · "구매하기" 버튼
- (실험 단계에서 확장) 컬러 variant 버튼, 스펙 카드, 프리미엄 톤 변경

### 5-1. 실험용 폴더 만들기

> **왜 폴더가 따로인가**: 앞서 설치한 `ui-poc-workspace/` 는 다음 워크숍에서 본격 UI PoC 를 만들 작업 폴더입니다. 오늘 실습은 **별도의 실험장** 을 따로 두어, 다음 워크숍의 작업 폴더를 깨끗하게 유지합니다.

**Mac** — 💻 터미널 에서:

```bash
# 💻 터미널 (Mac)
mkdir -p ~/Desktop/headset-lab && cd ~/Desktop/headset-lab
```

**Windows** — 💻 PowerShell 에서:

```powershell
# 💻 PowerShell (Windows)
$desktop = [Environment]::GetFolderPath('Desktop')
New-Item -ItemType Directory -Path "$desktop\headset-lab" -Force
cd "$desktop\headset-lab"
```

바탕화면에 `headset-lab` 폴더가 생긴 것을 파일 탐색기 / Finder 에서 확인해주세요.

### 5-2. 헤드셋 제품 이미지 준비

제공되는 헤드셋 제품 이미지를 `headset-lab` 폴더로 복사합니다. 이미지는 §2 에서 설치한 작업 폴더 안에 이미 포함되어 있습니다.

**이미지 원본 위치**: `바탕화면/ui-poc-workspace/docs/workshop-assets/headset.png`

**복사하는 방법**:

1. 파일 탐색기(Windows) / Finder(Mac) 를 **두 개** 엽니다.
2. 왼쪽 창: `바탕화면 → ui-poc-workspace → docs → workshop-assets` 폴더를 엽니다. `headset.png` 파일이 보입니다.
3. 오른쪽 창: `바탕화면 → headset-lab` 폴더를 엽니다.
4. `headset.png` 파일을 왼쪽에서 오른쪽으로 **드래그앤드롭** 합니다. (잘라내기가 아니라 **복사** 가 되도록, 드래그 중에 **Ctrl 키(Windows)** 또는 **Option 키(Mac)** 를 누르고 있으면 확실합니다.)

완료 후 폴더 상태:

```
바탕화면/headset-lab/
└── headset.png        ← 제공된 제품 이미지
```

> **이미지를 못 찾겠다면**: 💻 터미널에서 아래 한 줄로도 복사할 수 있습니다.
>
> ```bash
> # 💻 터미널 (Mac)
> cp ~/Desktop/ui-poc-workspace/docs/workshop-assets/headset.png ~/Desktop/headset-lab/
> ```
>
> ```powershell
> # 💻 PowerShell (Windows)
> Copy-Item "$([Environment]::GetFolderPath('Desktop'))\ui-poc-workspace\docs\workshop-assets\headset.png" -Destination "$([Environment]::GetFolderPath('Desktop'))\headset-lab\"
> ```

### 5-3. 첫 요청 — 원샷으로 3D 헤드셋 카탈로그 만들기

`headset-lab` 폴더에서 Claude Code 를 실행합니다. (이전 `ui-poc-workspace` 에서 열어둔 Claude Code 는 `/exit` 로 닫고 새로 여세요.)

```bash
# 💻 터미널 (Mac)
cd ~/Desktop/headset-lab && claude
```

```powershell
# 💻 PowerShell (Windows)
cd "$([Environment]::GetFolderPath('Desktop'))\headset-lab"; claude
```

§1 에서 정해둔 **가상 브랜드 이름과 슬로건** 을 대괄호 부분에 넣어, 💬 Claude Code 대화창에 아래 프롬프트를 입력합니다:

```
💬 Claude Code
이 폴더의 headset.png 를 사용해서, 가상 오디오 브랜드의 3D 헤드셋 카탈로그 페이지를
단일 index.html 파일로 만들어줘. Three.js 는 CDN 으로 불러오고, 파일을 더블클릭으로 열었을 때 바로 동작해야 해.

요구 사항:
- 중앙에 헤드셋 이미지가 3D 공간에서 천천히 회전하고, 마우스로 드래그해서 각도를 바꿀 수 있어야 해.
- 배경은 고급스러운 다크 그라디언트.
- 하단에 브랜드명, 제품명, 가격, "구매하기" 버튼을 배치해줘.
- 브랜드명은 "[본인이 정한 브랜드명]", 슬로건은 "[본인이 정한 슬로건]".
- 제품명과 가격은 프리미엄 헤드셋에 어울리게 자연스럽게 정해줘.
```

**파일 생성 승인 요청이 뜹니다.** 1번 **Yes** 로 허용하세요. (§4-2 에서 연습한 것과 같은 요청입니다)

완성되면 파일 탐색기 / Finder 에서 `headset-lab/index.html` 을 **더블클릭** 해서 브라우저에서 확인합니다. 헤드셋이 회전하고 마우스 드래그로 각도를 바꿀 수 있으면 성공.

> **브라우저 화면이 비어 있거나 이상하다면**: 아래 "트러블슈팅" 섹션을 확인하거나, 그대로 Claude 에게 "지금 브라우저가 까맣게 나와. 고쳐줘" 라고 말하면 수정해줍니다.

### 5-4. 실험: 요청을 점점 모호하게 해보기

같은 대화창에서 아래 네 가지 요청을 **순서대로** 입력합니다. 각 단계 후 `index.html` 을 브라우저에서 **새로고침(F5 또는 Cmd+R)** 해서 결과를 확인하세요.

| 단계      | 요청 예시                                                                                                 |
| --------- | --------------------------------------------------------------------------------------------------------- |
| 구체      | "제품 아래에 컬러 variant 버튼 3개(검정/화이트/그린)를 만들고, 누르면 CSS hue-rotate 로 색이 바뀌게 해줘" |
| 중간      | "우측에 스펙 정보(배터리 수명, 무게, 노이즈 캔슬링) 를 카드 형태로 배치해줘"                              |
| 모호      | "좀 더 프리미엄 브랜드처럼 보이게 해줘"                                                                   |
| 매우 모호 | "이 페이지를 보고 당장 구매 버튼을 누르고 싶게 만들어줘"                                                  |

각 단계에서 AI 가 **내 의도를 어떻게 해석했는지** 를 관찰하세요.

**결과가 마음에 안 들 때**: 다음 단계로 가기 전에 아래 프롬프트로 되돌릴 수 있습니다.

```
💬 Claude Code — 되돌리기
방금 변경은 취소하고 이전 상태로 돌려줘.
```

> **학습 포인트**: 모호한 요청일수록 AI 의 해석 편차가 커집니다. "프리미엄 브랜드처럼" 이 참가자마다 전혀 다른 결과로 나오는 것을 볼 수 있습니다. 이 체감이 **다음 워크숍의 PRD 작성 시 "왜 명세가 필요한가"** 의 근거가 됩니다.

### 5-5. 공유하기

완성된 `index.html` 의 화면을 **스크린샷** 으로 찍어 진행자가 안내한 공유 채널(단톡방 또는 Airtable 갤러리) 에 올리세요. 다음 워크숍의 쇼&텔 재료가 됩니다. 가상 브랜드명도 함께 적어주세요 — 같은 제품 이미지로 참가자마다 얼마나 다른 결과가 나왔는지 비교해볼 수 있습니다.

**스크린샷 단축키**:

- **Mac**: Shift + Cmd + 4 → 드래그로 영역 선택
- **Windows**: Win + Shift + S → 드래그로 영역 선택

실습이 끝났으면 💬 Claude Code 대화창을 `/exit` 로 종료하고 💻 터미널로 돌아옵니다.

## 6. Quick Research 로 리서치 PDF 받기 (16:10 ~ 16:50)

다음 워크숍에서 PRD 의 입력 재료가 될 **리서치 PDF** 를 받아둡니다. Amazon Quick Suite 의 **Research Agent** 기능을 사용하며, 이 섹션은 **웹 브라우저에서만** 진행합니다(터미널/Claude Code 는 사용하지 않습니다).

### 6-1. Amazon Quick Suite 접속

진행자가 공유한 접속 방법을 따릅니다. 일반적으로 아래 두 가지 중 하나입니다.

**Option A — AWS Workshop Studio 를 통한 접속 (권장)**:

1. 웹 브라우저에서 진행자가 공유한 **Workshop Studio 링크** 를 엽니다.
2. **Event Access Code** 를 입력합니다.
3. 좌측 메뉴의 **"Open AWS Console"** 클릭 → AWS 콘솔로 이동.
4. AWS 콘솔 상단 검색창에 `Quick Suite` 입력 → **Amazon Quick Suite** 선택.

**Option B — 본인 AWS 계정으로 접속**:

1. [https://console.aws.amazon.com/](https://console.aws.amazon.com/) 로그인.
2. 상단 검색창에 `Quick Suite` 입력 → **Amazon Quick Suite** 선택.

> **리전 확인**: Quick Suite 상단에 **us-east-1 (버지니아 북부)** 리전이 선택되어 있는지 확인하세요. 다른 리전이면 우측 상단에서 변경합니다.

### 6-2. Research Agent 실행

이 단계는 AWS 공식 가이드를 그대로 따라갑니다.

> **따라할 공식 가이드**: [Amazon Quick Suite Workshop — Research Agent 사용하기](https://catalog.workshops.aws/amazon-quick-suite-workshop/ko-KR/0400-research-agent/0410-use-research-agent)
>
> 위 링크를 브라우저에서 새 탭으로 열어두고, 아래 §6-3 의 프롬프트를 해당 가이드의 "입력 프롬프트" 위치에 붙여넣어 사용하세요.

1. Quick Suite 좌측 메뉴에서 **Research** (또는 "Research Agent") 를 클릭.
2. 공식 가이드의 안내에 따라 **새 리서치 세션** 을 시작합니다.

### 6-4. PDF 다운로드

### 3D 헤드셋 카탈로그 실습

브라우저 화면이 이상할 때 **Claude 에게 그대로 말하면** 대부분 해결됩니다. 자주 쓰는 프롬프트:

| 증상                            | Claude 에게 이렇게 말해보세요                                     |
| ------------------------------- | ----------------------------------------------------------------- |
| 브라우저 화면이 까맣기만 함     | "조명(light) 이 없는 것 같아. 기본 조명을 추가해줘"               |
| 헤드셋이 너무 작거나 안 보임    | "카메라 위치를 조정해서 헤드셋이 화면 중앙에 꽉 차게 해줘"        |
| 이미지가 안 뜨고 회색만 나옴    | "headset.png 경로가 제대로 적용됐는지 확인하고 다시 로드해줘"     |
| 마우스로 회전이 안 됨           | "OrbitControls 를 추가해서 마우스 드래그로 회전 가능하게 해줘"    |
| 헤드셋이 뒤집혀 있거나 기울어짐 | "모델의 초기 각도를 정면 기준으로 맞춰줘"                         |
| CDN 로드가 안 되고 빈 화면      | "three.js CDN URL 이 최신인지 확인하고 importmap 방식으로 바꿔줘" |
| 직전 수정이 마음에 안 들 때     | "방금 변경은 취소하고 이전 상태로 돌려줘"                         |
